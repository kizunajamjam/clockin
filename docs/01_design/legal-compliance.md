# 関連法令・労務コンプライアンス

最終更新: 2026-06-18

clockin が関係する主要な法令と、アプリとして対応すべき要件をまとめる。

---

## 1. 労働基準法（労基法）

### 1-1. 法定労働時間（第32条）

| 区分 | 上限 |
|---|---|
| 1日 | 8時間 |
| 1週 | 40時間（特例措置対象事業は44時間） |

**アプリへの影響**
- 月次集計画面で「法定時間外労働時間数」を表示する必要がある
- 1日8時間超・週40時間超の時間外労働時間を集計して給与計算に反映すること

### 1-2. 割増賃金（第37条）

| 区分 | 割増率（法定最低） |
|---|---|
| 法定時間外労働（月60時間以下） | 25%以上 |
| 法定時間外労働（月60時間超） | 50%以上（中小企業は2023年4月〜適用） |
| 法定休日労働 | 35%以上 |
| 深夜労働（22:00〜翌5:00） | 25%以上 |
| 深夜＋時間外の重複 | 50%以上 |

**アプリへの影響（重要）**
- 現在の設計では深夜割増（×1.25）のみ実装予定
- **時間外割増・休日割増は未実装 → 要追加**
- 「時間外計算はアプリ外で行ってください」と明示するか、実装するかを決める必要あり
- → **MVP方針：時間外・休日割増は実装しない。給与計算画面に「時間外・休日割増は別途ご確認ください」と注意書きを表示する**

### 1-3. 休憩時間（第34条）

| 労働時間 | 最低休憩 |
|---|---|
| 6時間超 | 45分 |
| 8時間超 | 1時間 |

**アプリへの影響**
- 打刻で入力した `break_minutes` が上記を下回っている場合に警告を表示すると親切
- ただし強制はしない（実態に合わせて入力させる）

### 1-4. 賃金台帳（第108条・第109条）

使用者は賃金台帳を作成し、**3年間保存**する義務がある。

法定記載事項：
1. 労働者氏名
2. 性別
3. 賃金計算期間
4. 労働日数
5. 労働時間数
6. 時間外・休日・深夜労働時間数（各区分）
7. 基本賃金
8. 各種手当の種類と金額
9. 控除項目と金額

**アプリへの影響**
- CSVエクスポートは上記の法定記載事項を網羅する必要がある
- データは最低3年間 Supabase に保持すること（削除機能に保存期間チェックを追加）
- 性別は現在のスキーマに未定義 → 任意項目として `staff` テーブルに追加を検討

### 1-5. 労働時間の客観的把握（2017年ガイドライン）

「労働時間の適正な把握のために使用者が講ずべき措置に関するガイドライン」により、
使用者はタイムカード・ICカード等の客観的方法で労働時間を記録・管理する義務がある。

**アプリへの影響**
- 打刻記録の改ざん防止: `attendances` テーブルに `created_at` と `updated_at` を記録済み
- 管理者が打刻を修正した場合は `note`（修正理由）を必須にすることを推奨
- 修正履歴の保持を検討（audit log）

## 2. 働き方改革関連法（2019年4月施行）

### 2-1. 時間外労働の上限規制

| 区分 | 上限 |
|---|---|
| 通常 | 月45時間・年360時間 |
| 臨時的な特別事情（36協定特別条項） | 月100時間未満・年720時間以下 |

**アプリへの影響**
- 月次集計で時間外労働が45時間に近づいたらオーナーに警告を出すと付加価値になる
- → **MVP後に実装する機能候補**

## 3. 最低賃金法

都道府県ごとに地域別最低賃金が定められており、これを下回る時給設定は違法。

**アプリへの影響**
- スタッフの時給設定時に、店舗の都道府県の最低賃金を下回っていたら警告を表示
- 最低賃金は毎年10月頃改定されるため、ハードコードは不可
- → **実装方針：最低賃金テーブルを DB に持つか、外部APIを利用。MVP後に対応**
- **MVP：注意書き「設定した時給が地域の最低賃金を下回っていないか確認してください」を表示**

## 4. 個人情報保護法

### 4-1. 取り扱う個人情報

| 情報 | 該当 | 備考 |
|---|---|---|
| 氏名・メールアドレス | 個人情報 | |
| 出退勤記録 | 個人情報 | |
| 給与情報 | 個人情報（要配慮情報に近い） | |
| GPS位置情報 | 個人情報 | 要同意取得 |

### 4-2. 対応事項

- **プライバシーポリシー**の作成・公開（リリース前必須）
- GPS位置情報取得の**目的・範囲を事前に明示し、同意を取得**すること
  - 打刻画面で初回のみ「位置情報は打刻時の不正防止のみに使用します」を表示
  - ブラウザのGPS許可ダイアログ前に説明文を出す
- データの第三者提供禁止・適切な安全管理措置（Supabase RLSで対応済み）
- 保有個人データの開示・訂正・削除に応じる仕組みが必要

## 5. 電子帳簿保存法

電子的に作成・保存する帳簿類（賃金台帳等）は以下を満たす必要がある：

- 真実性の確保（改ざん防止）
- 可視性の確保（検索可能・出力可能）

**アプリへの影響**
- Supabase の RLS + `updated_at` トリガーで改ざん防止は一定程度担保
- CSVエクスポート機能で可視性は確保
- 完全準拠には**タイムスタンプ付与**等が必要だが、MVP では対象外とし注意書きを表示

## 6. アプリとして対応すること（優先度別）

### MVP 必須

| 対応 | 根拠法令 |
|---|---|
| 深夜割増（22:00〜5:00 ×1.25）の実装 | 労基法37条 |
| 打刻記録の修正時に修正理由を必須入力にする | 労働時間把握ガイドライン |
| CSVに労基法108条の法定記載事項を含める | 労基法108条 |
| GPS取得前に利用目的を明示・同意を取る | 個人情報保護法 |
| データ削除時に3年保存期間のチェック | 労基法109条 |
| 給与計算画面に「時間外・休日割増は別途確認を」注意書き | 労基法37条 |
| 最低賃金未満の時給設定に警告メッセージ | 最低賃金法 |
| プライバシーポリシー・利用規約の作成 | 個人情報保護法 |

### MVP後に実装

| 対応 | 根拠法令 |
|---|---|
| 時間外割増（週40時間超・日8時間超）の自動計算 | 労基法37条 |
| 月60時間超の時間外に50%割増 | 労基法37条 |
| 時間外45時間接近アラート | 働き方改革法 |
| 最低賃金テーブルの年次更新 | 最低賃金法 |
| 修正履歴（audit log）の保持 | 労働時間把握ガイドライン |
| 有給休暇管理 | 労基法39条 |

## 7. 免責事項について

アプリはあくまでも「記録・計算補助ツール」であり、法令に基づく最終的な判断は
利用者（使用者）の責任において行われる旨を利用規約に明記する。

特に以下は利用規約・画面上で明示すること：
- 時間外・休日割増はアプリ外での確認が必要な場合がある
- 最低賃金の確認は利用者の責任
- 本アプリの計算結果は法的助言ではない
